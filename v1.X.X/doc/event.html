Events

Event Propagation

Note All events are propagated to and from the element to the darkmode toggle.You should listen to events from the HTML element directly rather than look for custom events.

> 
<div id="event-propagation-example">
    <button class="btn btn-primary" id="darkmodeEvent" data-plugin="bsdarkmode" data-root="#event-propagation-example"></button>
</div>
<script>
    $(function () {
        $('#darkmodeEvent').change(function () {
            log('Mode changed!')
        })
    })
</script>

Stopping Event Propagation

Passing true to the light, dark and toggle methods will enable the silent option to prevent the control from propagating the change event in cases where you want to update the controls light/dark state, but do not want to fire the onChange event.

> 
<div id="event-stop-example">
    <button class="btn btn-primary" data-plugin="bsdarkmode" id="darkmodeChange" data-root="#event-stop-example"></button>
    <button class="btn btn-success" onclick="$('#darkmodeChange').bsDarkmode('light',true)" >Light by API (silent)</button>
    <button class="btn btn-success" onclick="$('#darkmodeChange').bsDarkmode('dark',true)">Dark by API (silent)</button>
    <button class="btn btn-warning" onclick="$('#darkmodeChange').bsDarkmode('light')">Light by API (not silent)</button>
    <button class="btn btn-warning" onclick="$('#darkmodeChange').bsDarkmode('dark')">Dark by API (not silent)</button>
</div>
<script>
    $(function() {
    $('#darkmodeChange').change(function() {
        log('Mode changed!')
    })
    })
</script>
For ECMAS JavaScript replace $('#darkmodeChange')... with document.querySelector('#darkmodeChange')...
